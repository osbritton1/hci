static bool get_changing_orbitals(const size_t *exc_list, size_t exc_order, const size_t *occ_list, size_t nocc, ExcResult *res, size_t *new_occ_list)

Given a list of excitation orbitals, attempts to excite the provided configuration; on success, outputs the detailed information of the excitation and generates a new occupancy list.

To be explicit, this function returns true exactly when the the intersection of exc_list and occ_list and the set difference exc_list\ occ_list are both of size exc_order, in which case the intersection will be written to res->old_orbs, the difference to res->new_orbs and the excited configuration formed by substituting old_orbs with new_orbs to new_occ_list. This is essentially a customized merge algorithm for sorted lists.

Parameters:

  • exc_list – [in] Pointer to the list of orbitals involved in the excitation of length 2*exc_order (sorted in asc. order)

  • exc_order – [in] The order of the excitation

  • occ_list – [in] Pointer to the occupancy list of the reference determinant of length nocc (sorted in asc. order)

  • nocc – [in] The number of orbitals occupied in the reference determinant

  • res – [out] Pointer to the ExcResult to be written to on success

  • new_occ_list – [out] Pointer to array where the new occupied orbitals should be stored (of length nocc); will be sorted in asc. order as well

Returns:

True if the given excitation and determinant are compatible; when false, res and new_occ_list are garbage

  • add_list – [out] Pointer to Rank list storing configurations to be added

  • config_info – [in] Pointer to ConfigInfo object needed to perform unranking, control loop structure, etc.
