double get_diag_value(const size_t *occ_a, const size_t *occ_b, const ConfigInfo *config_info, const HCore *h1e, const ERITensor *eri_mo)

Calculates the expectation value of the Hamiltonian for a given configuration (i.e. the on-diagonal element).

Formula:

\[\begin{split}\begin{aligned} \langle\Psi|\Psi\rangle=&\sum\limits_a [a|h_\text{core}|a] + \dfrac{1}{2}\sum\limits_{ab} [aa|bb]-[ab|ba]&\\ =\,&\sum\limits_a (a|h_\text{core}|a) + \sum\limits_\bar{a} (\bar{a}|h_\text{core}|\bar{a})&\text{From $\verb|h1e_aa|$ and $\verb|h1e_bb|$}\\ +\,&\sum\limits_{a<b} (aa|bb)-(ab|ba)&\text{From $\verb|eri_mo_aaaa_s8|$}\\ +\,&\sum\limits_{\bar{a}<\bar{b}} (\bar{a}\bar{a}|\bar{b}\bar{b})-(\bar{a}\bar{b}|\bar{b}\bar{a})&\text{From $\verb|eri_mo_bbbb_s8|$}\\ +\,&\sum\limits_{a\bar{b}} (aa|\bar{b}\bar{b})&\text{From $\verb|eri_mo_aabb_s4|$} \end{aligned}\end{split}\]

Formula conventions:

  • \([\quad|\quad]\) are used for spin orbital summations, while \((\quad|\quad)\) are used for spatial orbital summations

  • \(a\) and \(b\) index occupied spin orbitals in \(\Psi\) if \([\quad|\quad]\) are used and occupied spatial orbitals if \((\quad|\quad)\) are used

  • When summing over spatial orbitals, \(a\) and \(b\) are \(\alpha\) MOs while \(\bar{a}\) and \(\bar{b}\) are \(\beta\) MOs

Parameters:

  • occ_a – [in] Pointer to the list of occupied \(\alpha\) orbitals

  • occ_b – [in] Pointer to the list of occupied \(\beta\) orbitals

  • config_info – [in] Pointer to ConfigInfo object needed to perform unranking, control loop structure, etc.

  • h1e – [in] Pointer to HCore object storing locations of the core Hamiltonian matrix elements

  • eri_mo – [in] Pointer to ERITensor object storing locations of the electron repulsion integrals

Returns:

The expectation value of the energy of the configuration

Calculates the matrix element between configurations that differ by one \(\alpha\) orbital.

Formula:

\[\begin{split}\begin{aligned} \langle\Psi|\Psi_a^r\rangle=&[a|h_\text{core}|r] + \sum\limits_{b} [ar|bb]-[ab|br]&\\ =\,&(a|h_\text{core}|r) &\text{From $\verb|h1e_aa|$}\\ +\,&\sum\limits_{b} (ar|bb)-(ab|br)&\text{From $\verb|eri_mo_aaaa_s8|$}\\ +\,&\sum\limits_{\bar{b}} (ar|\bar{b}\bar{b})&\text{From $\verb|eri_mo_aabb_s4|$} \end{aligned}\end{split}\]

Formula conventions:

  • \([\quad|\quad]\) are used for spin orbital summations, while \((\quad|\quad)\) are used for spatial orbital summations

  • \(\Psi_a^r\) differs from \(\Psi\) in the substitution of \(\alpha\) orbital \(a\) for \(r\)

  • \(b\) indexes occupied spin orbitals in \(\Psi\) if \([\quad|\quad]\) is used and occupied spatial orbitals if \((\quad|\quad)\) is used

  • When summing over spatial orbitals, \(b\) is an \(\alpha\) MO while \(\bar{b}\) is a \(\beta\) MO

  • The above formula assumes the configurations are in maximum coincidence; an extra sign factor derived from the indices of the changing orbitals in the old and new sorted occupation lists is necessary, which is calculated in the computation of single_exc

Parameters:

  • single_exc – [in] Pointer to ExcResult describing occupied and excitation orbitals

  • occ_a – [in] Pointer to the list of occupied \(\alpha\) orbitals

  • occ_b – [in] Pointer to the list of occupied \(\beta\) orbitals

  • config_info – [in] Pointer to ConfigInfo object needed to perform unranking, control loop structure, etc.

  • h1e – [in] Pointer to HCore object storing locations of the core Hamiltonian matrix elements

  • eri_mo – [in] Pointer to ERITensor object storing locations of the electron repulsion integrals

Returns:

The single excitation matrix element

double get_double_exc_value_from_store(const DoubleExcEntry *exc_entry, const ExcResult *double_exc)

Extracts a double excitation matrix element from storage in a DoubleExcEntry.

Given the list of occupied/old and excitation/new orbitals, first swaps the lowest-numbered orbital \(i\) into position using the symmetries of the ERI tensor, then determines the ordering of the remaining orbitals \(j\), \(k\), and \(l\), where \(i<j<k<l\). Once \(j\), \(k\), and \(l\) are located, the correct stored value can be extracted. As with get_double_exc_value_aa and get_double_exc_value_bb, an additional sign factor is necessary, handled in the computation of exc_entry.

Parameters:

  • exc_entry – [in] Pointer to DoubleExcEntry storing the excitation matrix element

  • double_exc – [in] Pointer to ExcResult describing occupied and excitation orbitals

Returns:

The stored double excitation matrix element

double get_mixed_exc_value(const ExcResult *single_exc_a, const ExcResult *single_exc_b, const ERITensor *eri_mo)

Calculates the matrix element between configurations that differ by one \(\alpha\) and one \(\beta\) orbital.

Formula:

\[\begin{split}\begin{aligned} \langle\Psi|\Psi_{a\bar{b}}^{r\bar{s}}\rangle=&[ar|\bar{b}\bar{s}]&\\ =\,&(ar|\bar{b}\bar{s}) &\text{From $\verb|eri_mo_aabb_s4|$}\\ \end{aligned}\end{split}\]

Formula conventions:

  • \([\quad|\quad]\) are used for spin orbitals, while \((\quad|\quad)\) are used for spatial orbitals

  • \(\Psi_{\bar{a}\bar{b}}^{\bar{r}\bar{s}}\) differs from \(\Psi\) in the substitution of \(\alpha\) orbital \(a\) for \(r\) and \(\beta\) orbitals \(\bar{b}\) and \(\bar{s}\)

  • The above formula assumes the configurations are in maximum coincidence; an extra sign factor derived from the indices of the changing orbitals in the old and new sorted occupation lists is necessary, which is calculated in the computation of single_exc_a and single_exc_b

Parameters:

  • single_exc_a – [in] Pointer to ExcResult describing occupied and excitation \(\alpha\) orbitals

  • single_exc_b – [in] Pointer to ExcResult describing occupied and excitation \(\beta\) orbitals

  • eri_mo – [in] Pointer to ERITensor object storing locations of the electron repulsion integrals

Returns:

The mixed excitation matrix element
